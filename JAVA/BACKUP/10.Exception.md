
# Exception Handling

## What is an Exception?

An exception is an event which is raised during the execution of the program and which interrupts the normal flow of the program.

or

When any unwanted or unexpected event is raised during the execution of the program and due to which the smooth execution of the program is interrupted, such event is called an **exception**.

**Event** means action.

## What is Exception Handling?

Exception handling is a mechanism of providing an alternate path for the program in case of exception so that normal flow of the program should not be interrupted.

## Paths of Execution

A program has 2 paths of execution:

1. Normal path of execution
2. Alternate path of execution

A program follows normal path of execution if there is no exception.

A program follows alternate path of execution if there is an exception and if we provide an alternate path.

## Normal Termination

If the program is executing completely and terminating, then it is called **normal termination**.

## Abnormal Termination

If the program is not executing completely and terminating, then it is called **abnormal termination**.

### Note

The aim of exception handling is graceful termination of the program.

---

## Difference Between Exception and Error

Exceptions are caused due to the Java programs and exceptions are recoverable.

Errors are caused due to the lack of system resources and errors are not recoverable.

```text
error     -> a mistake in the program
exception -> exception is also an error, but runtime error
```

### Notes

1. All exceptions are runtime errors.
2. In Java, all exceptions and errors are classes.
3. `Throwable` is a class and `Throwable` is the superclass or parent class for all the exceptions and errors in Java.

`Throwable` class has 2 direct subclasses or child classes:

1. `Exception`
2. `Error`

![Exception Handling Hierarchy](./images/ExceptionHandling.png)

`Exception` is a class and it has multiple subclasses or child classes which are all exceptions.

`Error` is a class and it has multiple subclasses or child classes which are all errors.

For all exceptions, `Exception` is the superclass or parent class.

For all errors, `Error` is the superclass or parent class.

## Types of Exceptions

There are 2 types of exceptions:

1. Checked exceptions
2. Unchecked exceptions

The exceptions which are checked or reported by the compiler at compilation time for smooth execution of the program at runtime and which are recoverable are called **checked exceptions**.

The exceptions which are not checked or not reported by the compiler at compilation time, which are raised directly at runtime and which are not recoverable are called **unchecked exceptions**.

`RuntimeException` and all the subclasses of `RuntimeException` are unchecked exceptions and remaining all are checked exceptions.

`Error` and all the subclasses of `Error` are unchecked.

## Checked Exceptions

Checked exceptions are 2 types:

1. Partially checked
2. Fully checked

If parent is checked, some children are checked and some children are unchecked, such exceptions are called **partially checked**.

Example: `Exception` and `Throwable`

If parent is checked and all children are also checked, such exceptions are called **fully checked**.

Example: `IOException`

## Exception Handling Blocks

In exception handling, to handle the exceptions we use `try`, `catch`, and `finally` blocks.

### try Block

It is used to write the risky code.

### catch Block

It is used to write the handling code.

### finally Block

It is used to write the cleanup code.

## Ways to Handle Checked Exceptions

There are 2 ways to handle checked exceptions:

1. Using `try` and `catch` blocks
2. Using `throws` keyword

## throws

`throws` is a keyword in Java and `throws` keyword is used to declare an exception.

Writing `throws` exception to the method is called declaring an exception.

If we do not want to handle checked exception using `try` and `catch` blocks and we want to delegate or pass the responsibility of handling the checked exception to the caller of the method, use `throws` keyword to declare an exception to the method.

`throws` keyword is used to declare checked exceptions but not unchecked exceptions.

## Ways to Handle Unchecked Exceptions

There is only one way to handle unchecked exception:

1. Using `try` and `catch` blocks

## Notes

1. If there is an exception in `try` block and appropriate `catch` block is available, then normal termination of the program will happen.

```java
try {
    int a = 10 / 0;
} catch (ArithmeticException e) {
}
```

2. If there is an exception in `try` block and appropriate `catch` block is not available, then abnormal termination of the program will happen.

```java
try {
    int a = 10 / 0;
} catch (ArrayIndexOutOfBoundsException e) {
}
```

3. If there is an exception outside `try` block, always abnormal termination of the program will happen.

## Methods to Display Exception Information

Sun Microsystems has provided 3 methods to display exception information.

All the 3 methods are available in `Throwable` class.

### 1. `void printStackTrace()`

Class: `Throwable`

It displays 3 pieces of information:

1. Name of the exception
2. Description
3. Stack trace

### 2. `String getMessage()`

Class: `Throwable`

It displays only 1 piece of information:

1. Description

### 3. `String toString()`

Class: `Throwable`

It displays 2 pieces of information:

1. Name of the exception
2. Description

## throw

`throw` is a keyword in Java. If we want to create exception object and throw explicitly from a method, use `throw` keyword.

```java
throw exceptionObject;
```

Using `throw` keyword, we can throw both checked exceptions and unchecked exceptions.

## try with Multiple catch Blocks

If multiple exceptions are there in a `try` block and on the occurrence of different exceptions if we want to execute different handling code, then we should use multiple `catch` blocks.

### Note

When we write `try` with multiple `catch` blocks, the order of `catch` blocks must be child to parent, but not parent to child.
